API Docs

로컬 서버의 게시물 목록 또는 상세 조회 엔드포인트에 동시 요청을 보내 캐시 경로와 DB 직접 경로 차이를 확인합니다.

어떤 게시물 상세 조회 API를 반복 호출할지 지정합니다. 예: 1이면 /posts/1을 테스트합니다.

동시에 몇 개의 워커가 요청을 보낼지 의미합니다. 값이 클수록 순간 부하가 커집니다.

각 워커가 같은 요청을 몇 번 반복할지 의미합니다. 총 요청 수는 Concurrency × Repeat 입니다.

같은 조건으로 캐시 경로와 DB 직접 경로를 비교하거나, 한쪽만 단독 실행할 수 있습니다.

목록 조회가 단건 조회보다 payload가 커서 보통 차이가 더 크게 보입니다.

예상 총 요청 수: 100

여러 명의 사용자가 여러 게시물 상세를 동시에 조회하는 상황을 재현합니다. 각 워커는 지정된 게시물 목록 전체를 순서대로 반복 호출합니다.

수동 모드에서 사용할 조회 대상 게시물 목록입니다. 쉼표로 구분합니다. 예: 1,2,3

DB 모드일 때 MongoDB에서 몇 개의 게시물 ID를 가져와 시나리오에 포함할지 의미합니다.

시뮬레이션에서 회전하며 사용할 가상 사용자 수입니다. 사용자별 인증은 없지만 서로 다른 사용자처럼 요청을 분산합니다.

동시에 실행할 워커 수입니다. 값이 높을수록 동시에 많은 사용자가 접속한 상황에 가깝습니다.

각 워커가 게시물 목록 전체를 몇 번 반복할지 의미합니다.

랜덤 모드일 때 워커가 한 번 순회할 때 생성할 랜덤 조회 요청 수입니다.

같은 조건으로 캐시 경로와 DB 직접 경로를 비교하거나, 한쪽만 단독 실행할 수 있습니다.

목록 조회는 DB에 더 많은 데이터를 읽게 하므로 차이가 더 크게 나는 편입니다.

예상 총 요청 수: 135
